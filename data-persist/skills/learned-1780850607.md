请为游戏项目创建完整的功能测试设计方案，按以下5个模块输出：

## 模块1: 测试策略
1. **测试金字塔**:
   - 单元测试 (60%): 核心逻辑函数独立测试
   - 集成测试 (30%): 模块间交互测试
   - E2E测试 (10%): 端到端用户流程测试

2. **覆盖率目标**:
   - 代码覆盖率: ≥80%
   - 分支覆盖率: ≥70%
   - 关键路径覆盖率: 100%

3. **测试环境**:
   - 测试框架: Vitest + Jest
   - 覆盖率工具: c8/istanbul
   - E2E工具: Playwright/Cypress

## 模块2: 核心玩法测试用例
按功能模块设计结构化测试用例:

| 测试ID | 模块 | 场景描述 | 输入数据 | 预期输出 | 边界条件 | 优先级 |
|--------|------|----------|----------|----------|----------|--------|
| TC-001 | 方块生成 | 正常生成验证 | levelConfig | 方块数=types×3 | 层数≤5 | P0 |
| TC-002 | 点击交互 | 可点击方块检测 | 方块位置 | true/false | 被遮挡判定 | P0 |
| TC-003 | 匹配检测 | 3个相同消除 | slot状态 | 消除动画+分数 | 不足3个 | P0 |
| ... | ... | ... | ... | ... | ... | ... |

设计至少20个测试用例，覆盖:
- 正常流程 (Happy Path)
- 边界条件 (Boundary)
- 异常场景 (Edge Case)
- 性能场景 (Performance)

## 模块3: 系统模块测试
1. **关卡系统**:
   - 关卡加载和初始化
   - 解锁机制验证
   - 进度保存/加载
   - 星级评价计算

2. **道具系统**:
   - 撤销道具功能和次数限制
   - 洗牌道具随机性验证
   - 移除道具效果验证
   - 道具冷却和恢复

3. **UI系统**:
   - 分数显示和更新
   - 动画效果流畅度
   - 响应式布局适配
   - 交互反馈及时性

4. **数据持久化**:
   - localStorage读写
   - 数据格式验证
   - 存储容量限制
   - 数据迁移兼容

## 模块4: 测试数据设计
1. **夹具数据**:
```javascript
// 标准测试关卡配置
const testLevel = {
  id: 1,
  tileTypes: 4,
  layers: 2,
  rows: 3,
  cols: 4,
  targetScore: 100
};

// 标准测试用户进度
const testUserProgress = {
  currentLevel: 1,
  levelStars: { 1: 0, 2: 0 },
  highScores: {},
  totalScore: 0
};
```

2. **Mock策略**:
   - 随机数种子固定
   - 时间函数Mock
   - 本地存储Mock
   - 动画帧Mock

3. **边界值数据**:
   - 最小配置: 1类型, 1层, 1×1网格
   - 最大配置: 12类型, 5层, 8×8网格
   - 空状态: 无方块, 满槽位
   - 极端分数: 0分, 溢出分

## 模块5: 验收标准
1. **功能验收清单**:
   - [ ] 所有P0测试用例通过
   - [ ] 核心玩法流程无阻断
   - [ ] 数据持久化正常
   - [ ] 道具系统功能完整
   - [ ] 关卡解锁逻辑正确

2. **性能指标**:
   - 首屏加载: <3秒
   - 帧率稳定: ≥55fps
   - 内存占用: <100MB
   - 响应延迟: <100ms

3. **兼容性要求**:
   - Chrome 90+, Safari 14+, Firefox 88+
   - iOS 14+, Android 10+
   - 屏幕尺寸: 320px-1920px

4. **测试报告格式**:
```markdown

- 通过数: X (XX%)

## 失败详情

|--------|------|----------|----------|

- 优化建议
```

请根据具体游戏项目调整测试用例和验收标准。
