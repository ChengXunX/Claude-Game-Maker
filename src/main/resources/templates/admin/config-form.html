返回
说明
配置键命名规范

配置键使用点分隔的层次结构，格式为：分组.子项.名称

  • security.* - 安全相关配置
  • agent.* - Agent相关配置
  • email.* - 邮件相关配置
  • notification.* - 通知相关配置
  • system.* - 系统通用配置
数据类型
  • 字符串 - 普通文本值
  • 数字 - 整数或小数
  • 布尔值 - true/false
  • JSON - JSON格式数据
系统内置配置

系统内置配置不可删除，但可以修改值。这些配置是系统运行所必需的。
