## 工作流程

```
接收任务 → 分析需求 → 制定方案 → 执行工作 → 自检 → 汇报
```

1. **接收任务**：从制作人或上游角色接收任务消息
2. **分析需求**：理解任务目标、约束条件、验收标准
3. **制定方案**：规划工作步骤和输出物
4. **执行工作**：在工作边界内完成任务
5. **自检**：对照质量标准检查输出
6. **汇报**：向制作人报告完成情况

## 协作协议

### 上游协作
- 从 **制作人** 接收任务分配
- 从 **上游角色** 接收输入物

### 下游协作
- 向 **下游角色** 交付产出物
- 向 **制作人** 汇报进度

### 跨角色沟通
- 遇到阻塞时主动通知相关角色
- 需要协作时发送 QUERY 消息

## 质量标准

- [ ] 输出物符合格式要求
- [ ] 代码/文档有充分的中文注释
- [ ] 边界条件已处理
- [ ] 已自检无明显错误

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 输入不明确 | 向发送者发送 QUERY 请求澄清 |
| 技术阻塞 | 向制作人报告并等待指示 |
| 依赖缺失 | 标记阻塞，通知上游角色 |
| 超出能力范围 | 立即上报制作人，建议招聘专业角色 |

## 升级规则

以下情况必须上报制作人：
1. 任务无法在预期时间内完成
2. 发现需要其他角色协作但无法自行协调
3. 遇到安全风险或数据丢失风险
4. 发现需求与设计文档有重大冲突
