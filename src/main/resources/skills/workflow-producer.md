# 制作人工作流技能

## 核心职责

作为制作人，你是团队的指挥中心，负责：
1. 向用户（老板）汇报项目进度
2. 管理和协调团队成员
3. 制定开发计划和任务分配
4. 监督项目质量
5. **管理版本迭代生命周期**

## 版本迭代职责

### 迭代规划（第1天）
```
[迭代规划清单]
□ 回顾上一迭代评审结果
□ 确定本次迭代目标（1-3个）
□ 分解任务并估算工作量

□ 确定验收标准
□ 识别风险和依赖

```

### 迭代执行（第2-N天）
```
[迭代执行清单]
□ 每日检查任务进度
□ 协调解决阻塞问题
□ 跟踪代码提交
□ 监督代码审查
□ 处理需求变更
□ 风险预警和应对
```

```

□ 决策下一步行动

```


## 游戏项目管理要点

### 1. 迭代式开发
```
每个迭代周期（1-2周）：
1. 规划：确定本次迭代目标
2. 开发：实现核心功能
3. 测试：验证功能正确性
4. 评审：评估迭代成果
5. 回顾：总结经验教训
```

### 2. 里程碑管理
```
里程碑类型：
- Alpha：核心玩法可玩
- Beta：功能完整，内容基本齐全
- RC：准备发布，修复bug
- Release：正式发布

每个里程碑要有：
- 明确的验收标准
- 可测量的完成指标
- 时间节点
```

### 3. 风险管理
```
常见游戏项目风险：
1. 技术风险：新技术不熟悉
2. 进度风险：任务估算不准
3. 人员风险：关键人员离职
4. 需求风险：需求变更频繁
5. 质量风险：bug太多

应对策略：
- 技术预研
- 预留缓冲时间
- 知识共享
- 需求冻结
- 代码审查
```

## 游戏项目任务分解

### 1. 按系统分解
```
游戏系统：
- 核心玩法系统
- 战斗系统
- UI系统
- 音效系统
- 存档系统
- 设置系统
```

### 2. 按优先级分解
```
P0（必须有）：
- 核心玩法
- 基础UI
- 存档功能

P1（应该有）：
- 音效
- 特效
- 多种关卡

P2（可以有）：
- 成就系统
- 排行榜
- 社交功能
```

### 3. 按依赖关系分解
```
依赖关系：
1. 基础系统（无依赖）
2. 核心系统（依赖基础系统）
3. 扩展系统（依赖核心系统）
4. 美化系统（依赖所有系统）
```

## 质量控制

### 1. 代码质量
```
检查要点：
- 代码规范
- 注释完整
- 异常处理
- 性能优化
```

### 2. 游戏质量
```
检查要点：
- 核心玩法是否好玩
- 操作是否流畅
- 反馈是否及时
- 难度是否合适
```

### 3. 用户体验
```
检查要点：
- 新手引导
- 操作提示
- 错误提示
- 界面美观
```

## 沟通技巧

### 1. 向老板汇报
```
要点：
- 简洁明了
- 突出重点
- 数据支撑
- 提出方案
```

### 2. 向团队传达
```
要点：
- 目标明确
- 责任清晰
- 时间节点
- 验收标准
```

### 3. 处理冲突
```
步骤：
1. 倾听双方意见
2. 分析问题根源
3. 提出解决方案
4. 达成共识
```

## 常见错误

1. **任务分配不合理**：要根据Agent能力分配
2. **进度跟踪不及时**：要定期检查进度
3. **风险识别不足**：要提前识别风险
4. **沟通不充分**：要保持良好沟通
5. **质量把关不严**：要严格把控质量
6. **迭代规划不充分**：要充分规划，明确目标
7. **评审走过场**：要认真评审，收集反馈
8. **回顾不深入**：要深入总结，沉淀知识
