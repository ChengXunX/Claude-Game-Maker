# 游戏 UI 设计技能

> 适用于：游戏内界面设计，包括 HUD、菜单、背包、商店、对话框、成就等
> 不适用于：游戏开发管理后台（那是管理界面，不是游戏 UI）

## 0. 设计意图推断

1. **游戏类型** - RPG / 射击 / 策略 / 休闲 / 模拟经营
2. **美术风格** - 像素 / 扁平 / 写实 / 卡通 / 赛博朋克
3. **目标平台** - PC / 移动端 / 主机 / 小程序
4. **交互方式** - 鼠标键盘 / 触屏 / 手柄

## 1. HUD 设计原则

### 信息分层
```
第一层（始终可见）：生命值、能量/弹药、小地图、任务目标
第二层（按需显示）：技能冷却、Buff/Debuff 图标、队友状态
第三层（按键触发）：详细属性、装备对比、任务日志

规则：
- 第一层信息占用屏幕面积不超过 15%
- 放在屏幕四角，不要遮挡中心视野
- 用半透明背景（opacity: 0.6-0.8）避免完全遮挡游戏画面
```

## 2. 菜单系统

### 主菜单
```
规则：
- 全屏背景（游戏场景渲染图或动态背景）
- 菜单项垂直排列在左侧或居中
- 选中项：放大 + 发光/高亮 + 描述文字展开
- 未选中项：半透明 + 缩小
- 过渡动画：菜单项交错入场（200ms 间隔）
- 按钮样式：不要用标准 HTML 按钮，用游戏风格的边框/底板
```

### 暂停菜单
```
规则：
- 半透明黑色遮罩覆盖游戏画面（opacity: 0.7）
- 菜单面板居中，带游戏风格边框
- 选项：继续游戏 / 设置 / 重新开始 / 退出
- 选中效果：边框发光 + 文字高亮
- 背景模糊（backdrop-filter: blur(4px)）
```

## 3. 背包/物品系统

### 网格背包
```
规则：
- 固定网格（如 8x6），每格 48-64px
- 物品图标居中，稀有度用边框颜色区分
  - 普通：灰色边框
  - 稀有：蓝色边框 + 微弱发光
  - 史诗：紫色边框 + 较强发光
  - 传说：金色边框 + 粒子特效
- 堆叠数量：右下角小数字
- 拖拽交互：拖起时物品放大 1.2x + 阴影加深
- Tooltip：悬浮 0.5s 后显示，包含属性对比
```

### 物品详情面板

## 4. 对话/剧情系统

### 对话框
```
规则：
- 底部 1/4 屏幕，半透明深色背景
- 左侧：角色头像（64-96px，圆形或圆角方形）
- 右侧：角色名 + 对话文字
- 文字逐字显示（打字机效果，30-50ms/字）
- 选项：垂直排列在对话框下方或内嵌
- 选项悬浮：背景变亮 + 左侧出现指示箭头
```

### 选项分支
```
规则：
- 选项数量：2-4 个最佳
- 每个选项一行，高度 44-48px
- 选中态：背景高亮 + 左侧竖线标记
- 有后果的选项：用图标提示（⚠️ 影响剧情）
- 倒计时选项：右侧显示进度条
```

## 6. 成就/解锁系统

```
规则：
- 成就弹出：从右上角滑入，停留 3-5 秒后滑出
- 弹出内容：图标 + 成就名 + 简短描述
- 背景：半透明 + 发光边框（根据稀有度）
- 音效配合：叮~的提示音
- 成就列表：网格或列表视图，已解锁全亮，未解锁灰色+问号
```

## 7. 游戏 UI 动效

### 原则
- 所有 UI 动效必须 < 300ms（游戏不能等）
- 使用缓出曲线：`ease-out` 或 `cubic-bezier(0, 0.55, 0.45, 1)`
- 数字变化：滚动计数（200ms）
- 血条变化：先快后慢（0.5s 走完 80%，1s 走完剩余 20%）
- 物品获取：从物品位置飞向背包图标（抛物线轨迹）

### 禁止
- 禁止超过 1 秒的过渡动画
- 禁止阻塞操作的 loading 动画
- 禁止闪烁/抖动效果（影响玩家注意力）
