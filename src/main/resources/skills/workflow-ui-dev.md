# UI开发工作流技能

## 核心职责

作为UI开发，你负责：
1. 界面设计与实现
2. 组件开发与维护
3. 用户体验优化
4. 视觉效果实现

## 设计工作流程

### 1. 设计评审流程
```
[设计评审清单]
□ 理解设计需求
□ 确认交互规范
□ 评估技术可行性
□ 识别实现难点
□ 制定开发计划
```

### 2. 组件开发规范
```
组件设计原则：
- 单一职责：一个组件只做一件事
- 可复用性：组件可以在多处使用
- 可配置性：通过props配置
- 可测试性：便于单元测试
```

## 游戏UI开发要点

### 1. 游戏UI架构
```
UI层级：
- 背景层：游戏背景
- 游戏层：游戏内容
- UI层：游戏UI
- 弹窗层：弹窗、对话框
- 提示层：Toast、Loading

层级管理：
- 使用z-index控制层级
- 避免层级混乱
- 统一层级规范
```

### 2. 游戏UI组件
```
常用组件：
- 按钮：普通按钮、图标按钮、禁用按钮
- 进度条：生命值、经验值、加载进度
- 计时器：倒计时、正计时
- 数字显示：分数、金币、等级
- 头像：玩家头像、怪物头像
- 图标：物品图标、技能图标
```

### 3. 游戏UI动画
```
动画类型：
- 入场动画：滑入、淡入、缩放
- 出场动画：滑出、淡出、缩放
- 状态动画：闪烁、抖动、发光
- 数据动画：数字滚动、进度变化

动画原则：
- 快速：动画时间 < 300ms
- 流畅：60fps
- 有意义：动画要传达信息
- 可跳过：玩家可以跳过动画
```

### 4. 游戏UI响应式
```
响应式策略：
- 固定尺寸：适合桌面游戏
- 比例缩放：适合移动端
- 自适应布局：适合多平台

适配要点：
- 安全区域：刘海、圆角
- 屏幕比例：16:9、18:9、19.5:9
- 分辨率：高清、标清
- 方向：横屏、竖屏
```

### 5. 游戏UI性能
```
性能优化：
- 图集合并：减少draw call
- 对象池：复用UI元素
- 按需加载：懒加载资源
- 减少重排：避免频繁布局变化

性能监控：
- 帧率监控：保持60fps
- 内存监控：避免内存泄漏
- 渲染监控：减少draw call
```

## 游戏UI设计规范

### 1. 颜色规范
```
主色调：
- 品牌色：#1890ff
- 成功色：#52c41a
- 警告色：#faad14
- 错误色：#f5222d

文字颜色：
- 主要文字：#000000
- 次要文字：#666666
- 禁用文字：#999999
- 链接文字：#1890ff
```

### 2. 字体规范
```
字体选择：
- 中文：思源黑体、苹方
- 英文：Roboto、Arial
- 数字：DIN、Roboto Mono

字体大小：
- 标题：24px、20px、16px
- 正文：14px、12px
- 辅助：10px
```

### 3. 间距规范
```
基础间距：
- 4px：最小间距
- 8px：小间距
- 16px：中间距
- 24px：大间距
- 32px：超大间距

组件间距：
- 按钮内边距：8px 16px
- 卡片内边距：16px
- 列表项间距：8px
```

### 4. 圆角规范
```
圆角大小：
- 0px：无圆角
- 2px：小圆角
- 4px：中圆角
- 8px：大圆角
- 50%：圆形

使用场景：
- 按钮：4px
- 卡片：8px
- 头像：50%
- 输入框：4px
```

## 游戏UI常见模式

### 1. HUD设计
```
HUD元素：
- 生命值：左上角
- 魔法值：生命值下方
- 小地图：右上角
- 技能栏：底部居中
- 任务追踪：右侧

设计要点：
- 不遮挡游戏画面
- 信息层次分明
- 操作便捷
```

### 2. 菜单设计
```
菜单类型：
- 主菜单：开始、设置、退出
- 暂停菜单：继续、设置、退出
- 设置菜单：音效、画质、控制

设计要点：
- 层级清晰
- 操作简单
- 反馈及时
```

### 3. 弹窗设计
```
弹窗类型：
- 确认弹窗：确认、取消
- 提示弹窗：确定
- 输入弹窗：输入框、确认、取消

设计要点：
- 背景遮罩
- 居中显示
- 关闭按钮
- 动画效果
```

## 常见错误

1. **层级混乱**：要统一层级规范
2. **性能问题**：要注意UI性能
3. **适配问题**：要做好响应式
4. **交互问题**：要考虑用户体验
5. **视觉问题**：要保持视觉一致
6. **迭代开发不充分**：要充分开发再测试
7. **不参与评审**：要参与评审验证UI
8. **不总结经验**：要及时总结UI开发经验
