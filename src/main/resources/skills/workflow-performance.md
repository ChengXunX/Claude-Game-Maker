# 性能优化技能

## 优化原则

### 核心理念
- **测量优先**: 先测量后优化
- **瓶颈定位**: 找到真正的瓶颈
- **权衡取舍**: 性能与可维护性平衡
- **持续监控**: 建立性能监控体系

## 优化流程

### 1. 性能分析
```
[性能分析步骤]
□ 收集性能数据
□ 分析性能瓶颈
□ 确定优化目标
□ 制定优化方案
□ 实施优化
□ 验证优化效果
```

### 2. 性能指标
```markdown
## 性能指标定义

### 响应时间
| 操作 | 目标 | 可接受 | 不可接受 |
|------|------|--------|----------|
| 页面加载 | < 1s | < 3s | > 3s |
| API响应 | < 200ms | < 500ms | > 500ms |
| 数据库查询 | < 50ms | < 100ms | > 100ms |

### 吞吐量
- QPS: 每秒查询数
- TPS: 每秒事务数
- 并发用户数

### 资源使用
- CPU使用率: < 70%
- 内存使用率: < 80%
- 磁盘IO: < 80%
- 网络带宽: < 70%
```

### 2. 缓存优化
```markdown
## 缓存策略

### 缓存层次
| 层次 | 技术 | 场景 |
|------|------|------|
| 浏览器缓存 | HTTP缓存头 | 静态资源 |
| CDN缓存 | CDN服务 | 全球加速 |
| 应用缓存 | Redis/Memcached | 热点数据 |
| 数据库缓存 | Query Cache | 查询结果 |

### 缓存策略
- Cache-Aside: 先查缓存再查数据库
- Write-Through: 写入时同时更新缓存
- Write-Behind: 异步写入缓存
- Read-Through: 缓存自动加载

### 缓存问题
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 缓存穿透 | 查询不存在数据 | 布隆过滤器/空值缓存 |
| 缓存击穿 | 热点key过期 | 互斥锁/永不过期 |
| 缓存雪崩 | 大量key同时过期 | 随机过期时间 |
```

## 代码优化技巧

### 算法优化
- 选择合适的算法
- 减少时间复杂度
- 减少空间复杂度

### 并发优化
```java
// 使用线程池
ExecutorService executor = Executors.newFixedThreadPool(10);
executor.submit(() -> {
    // 异步任务
});

// 使用CompletableFuture
CompletableFuture.supplyAsync(() -> {
    return fetchData();
}).thenApply(data -> {
    return process(data);
});
```

### 内存优化
- 避免内存泄漏
- 合理使用对象池
- 及时释放资源
- 使用弱引用/软引用
```
