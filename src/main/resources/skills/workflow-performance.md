# 性能优化技能

## 优化原则

### 核心理念
- **测量优先**: 先测量后优化
- **瓶颈定位**: 找到真正的瓶颈
- **权衡取舍**: 性能与可维护性平衡
- **持续监控**: 建立性能监控体系

## 优化流程

### 1. 性能分析
```
[性能分析步骤]
□ 收集性能数据
□ 分析性能瓶颈
□ 确定优化目标
□ 制定优化方案
□ 实施优化
□ 验证优化效果
```

### 2. 性能指标
```
前端指标：
- 首次加载时间（FCP）
- 首次交互时间（TTI）
- 帧率（FPS）
- 内存使用

后端指标：
- 响应时间
- 吞吐量
- 并发数
- 错误率

游戏指标：
- 帧率（目标60fps）
- 加载时间
- 内存使用
- 网络延迟
```

### 3. 告警机制
```
告警规则：
- 帧率低于30fps
- 加载时间超过5秒
- 内存使用超过512MB
- 错误率超过1%

告警方式：
- 邮件通知
- 短信通知
- 钉钉/飞书通知
```

## 优化案例

### 1. 帧率优化案例
```
问题：游戏帧率只有30fps
分析：渲染对象过多
方案：使用对象池 + 视锥剔除
结果：帧率提升到60fps
```

### 2. 加载优化案例
```
问题：游戏加载时间10秒
分析：资源未压缩
方案：压缩资源 + 按需加载
结果：加载时间降到3秒
```

### 3. 内存优化案例
```
问题：内存泄漏导致崩溃
分析：对象未销毁
方案：使用对象池 + 及时销毁
结果：内存稳定在256MB
```

## 常见错误

1. **盲目优化**：要先测量再优化
2. **过早优化**：要先保证功能正确
3. **优化过度**：要权衡性能和可维护性
4. **不监控**：要建立监控体系
5. **不测试**：要验证优化效果
6. **迭代性能不达标**：要确保迭代性能达标
7. **不参与评审**：要参与评审验证性能
8. **不总结经验**：要及时总结优化经验
