### 2. API 文档模板
```markdown
# API 文档

## 接口列表

### 获取资源列表

**请求**

```
GET /api/resources
```

**参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码 |
| size | int | 否 | 每页数量 |

**响应**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 100
  }
}
```

**错误码**

| 错误码 | 说明 |
|--------|------|
| 400 | 参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
```

### 2. 游戏API文档
```markdown
# 游戏API文档

## 玩家API

### 登录

**请求**

```
POST /api/player/login
```

**参数**

```json
{
  "username": "string",
  "password": "string"
}
```

**响应**

```json
{
  "code": 200,
  "data": {
    "token": "string",
    "player": {
      "id": 1,
      "name": "string",
      "level": 1
    }
  }
}
```

### 获取背包

**请求**

```
GET /api/player/inventory
```

**响应**

```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": 1,
        "name": "string",
        "count": 10
      }
    ]
  }
}
```
```

## 文档规范

### 1. 命名规范
```
- 文件名：小写，连字符分隔
- 标题：清晰简洁
- 代码块：标注语言
```

### 2. 格式规范
```
- 使用Markdown格式
- 使用表格展示数据
- 使用代码块展示代码
- 使用列表展示步骤
```

### 3. 内容规范
```
- 完整性：覆盖所有功能
- 准确性：信息准确无误
- 及时性：随代码更新
- 易读性：易于理解
```

## 常见错误

1. **文档不完整**：要覆盖所有功能
2. **文档过时**：要随代码更新
3. **文档不清晰**：要易于理解
4. **文档不准确**：要准确无误
5. **文档不维护**：要定期维护
