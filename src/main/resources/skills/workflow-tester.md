## 游戏测试要点

### 1. 功能测试
```
测试范围：
- 核心玩法：游戏基本功能
- 战斗系统：攻击、技能、死亡
- UI系统：界面显示、交互
- 存档系统：保存、加载、云存档
- 设置系统：音效、画质、控制

测试要点：
- 正常流程：按预期操作
- 异常流程：非法操作
- 边界条件：极端情况
- 兼容性：不同设备
```

### 2. 性能测试
```
测试指标：
- 帧率：目标60fps
- 内存：不超过限制
- 加载时间：不超过3秒
- 网络延迟：不超过100ms

测试工具：
- Chrome DevTools：性能分析
- Lighthouse：性能评分
- 自定义工具：游戏特定指标
```

### 3. 兼容性测试
```
测试维度：
- 浏览器：Chrome、Firefox、Safari、Edge
- 设备：PC、手机、平板
- 系统：Windows、macOS、iOS、Android
- 分辨率：不同屏幕尺寸

测试要点：
- 功能正常
- 显示正确
- 性能可接受
- 交互正常
```

### 4. 安全测试
```
测试内容：
- 输入验证：防止注入
- 权限控制：防止越权
- 数据安全：防止泄露
- 网络安全：防止攻击

测试工具：
- OWASP ZAP：安全扫描
- Burp Suite：渗透测试
- 自定义工具：游戏特定漏洞
```

### 5. 用户体验测试
```
测试维度：
- 易用性：操作是否简单
- 可学性：新手能否学会
- 满意度：玩家是否满意
- 可访问性：残障人士能否使用

测试方法：
- 用户访谈
- 可用性测试
- A/B测试
- 问卷调查
```

## 游戏测试用例示例

### 1. 登录测试
```
TC001: 正常登录
- 前置条件：已注册账号
- 测试步骤：
  1. 打开游戏
  2. 输入用户名和密码
  3. 点击登录
- 预期结果：登录成功，进入主界面

TC002: 密码错误
- 前置条件：已注册账号
- 测试步骤：
  1. 打开游戏
  2. 输入用户名和错误密码
  3. 点击登录
- 预期结果：提示密码错误
```

### 2. 战斗测试
```
TC003: 普通攻击
- 前置条件：进入战斗
- 测试步骤：
  1. 选择攻击目标
  2. 点击攻击按钮
- 预期目标：目标受到伤害

TC004: 技能释放
- 前置条件：进入战斗，技能可用
- 测试步骤：
  1. 选择技能
  2. 选择目标
  3. 释放技能
- 预期结果：技能效果生效
```

### 3. 存档测试
```
TC005: 保存游戏
- 前置条件：游戏中
- 测试步骤：
  1. 点击保存
  2. 确认保存
- 预期结果：保存成功

TC006: 加载存档
- 前置条件：有存档
- 测试步骤：
  1. 点击加载
  2. 选择存档
  3. 确认加载
- 预期结果：加载成功，恢复游戏状态
```

## 自动化测试

### 1. 单元测试
```javascript
// 测试伤害计算
describe('DamageCalculator', () => {
  it('should calculate basic damage', () => {
    const damage = calculateDamage(100, 50);
    expect(damage).toBe(50);
  });
  
  it('should apply critical hit', () => {
    const damage = calculateDamage(100, 50, true);
    expect(damage).toBe(100);
  });
});
```

### 2. 集成测试
```javascript
// 测试战斗流程
describe('BattleSystem', () => {
  it('should handle player attack', () => {
    const battle = new BattleSystem();
    battle.startBattle(player, enemy);
    
    player.attack(enemy);
    
    expect(enemy.hp).toBeLessThan(enemy.maxHp);
  });
});
```

### 3. 端到端测试
```javascript
// 测试登录流程
describe('Login Flow', () => {
  it('should login successfully', () => {
    cy.visit('/login');
    cy.get('#username').type('testuser');
    cy.get('#password').type('password');
    cy.get('#login-btn').click();
    cy.url().should('include', '/dashboard');
  });
});
```

## 常见错误

1. **测试覆盖不足**：要覆盖所有场景
2. **测试环境不一致**：要统一测试环境
3. **测试数据不真实**：要使用真实数据
4. **测试报告不详细**：要记录详细信息
5. **回归测试不足**：要定期回归测试
6. **迭代测试不充分**：要充分测试再评审
7. **不参与评审**：要参与评审验证质量
8. **不总结经验**：要及时总结测试经验
