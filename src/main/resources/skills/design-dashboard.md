# 仪表盘设计技能

> 适用于：Agent 监控、资源统计、系统健康、性能面板、流水线状态等数据密集页面
> 核心目标：信息密度高但不混乱，关键数据一目了然

## 0. 设计意图推断

先回答三个问题：
1. **数据类型** - 实时监控 / 历史统计 / 状态总览 / 告警面板
2. **查看频率** - 每天看一次 / 频繁盯着看 / 出问题才看
3. **决策场景** - 看完要做什么？启动 Agent / 调整配置 / 排查问题

## 1. 信息架构

### 信息层次
```
第一层（始终可见）：关键指标、状态总览
第二层（按需展开）：详细数据、历史趋势
第三层（点击查看）：原始数据、详细日志
```

### 布局原则
```
- 重要信息放在左上角
- 相关信息放在一起
- 使用卡片分组
- 保持视觉层次
```

## 2. 数据可视化

### 1. 数字展示
```
规则：
- 大数字突出显示
- 添加单位说明
- 使用颜色区分状态
- 添加趋势箭头
```

### 2. 图表选择
```
- 折线图：趋势变化
- 柱状图：数量对比
- 饼图：占比分析
- 仪表盘：进度展示
```

### 3. 状态指示
```
- 绿色：正常
- 黄色：警告
- 红色：错误
- 灰色：未知
```

## 4. 交互设计

### 1. 筛选过滤
```
- 时间范围选择
- 状态筛选
- 类型筛选
- 关键词搜索
```

### 2. 刷新机制
```
- 自动刷新：可配置间隔
- 手动刷新：刷新按钮
- 实时推送：WebSocket
```

### 3. 导出功能
```
- 导出CSV
- 导出PDF
- 截图分享
```

## 5. 响应式设计

### 1. 桌面端
```
- 多列布局
- 侧边栏导航
- 详细信息展示
```

### 2. 平板端
```
- 两列布局
- 折叠导航
- 简化信息
```

### 3. 移动端
```
- 单列布局
- 底部导航
- 核心信息
```

## 常见错误

1. **信息过载**：要控制信息密度
2. **层次不清**：要建立清晰的信息层次
3. **交互复杂**：要简化交互流程
4. **性能问题**：要优化数据加载
5. **响应式差**：要做好响应式适配
