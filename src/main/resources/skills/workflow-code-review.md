# 代码审查技能

## 审查原则

### 核心理念
- **建设性**: 提出改进建议而非批评
- **具体性**: 指出具体问题和解决方案
- **学习性**: 相互学习共同成长
- **效率性**: 及时审查不阻塞开发

## 审查流程

### 1. 审查准备
```
[审查准备清单]
□ 了解变更背景和目的
□ 阅读相关需求文档
□ 确认代码规范标准
□ 预留充足审查时间
```

## 审查标准

### 命名规范
| 元素 | 规范 | 示例 |
|------|------|------|
| 类名 | 大驼峰 | UserService |
| 方法名 | 小驼峰 | getUserById |
| 常量 | 全大写下划线 | MAX_RETRY_COUNT |
| 变量 | 小驼峰 | userName |

### 代码格式
```java
// 方法之间空一行
public void method1() {
    // 代码
}

public void method2() {
    // 代码
}

// 花括号换行
if (condition) {
    // 代码
} else {
    // 代码
}
```

### 注释规范
```java
/**
 * 类注释：说明类的用途
 * 
 * @author 作者
 * @since 版本
 */
public class Example {
    
    /** 字段注释：说明字段用途 */
    private String field;
    
    /**
     * 方法注释：说明方法功能
     * 
     * @param param 参数说明
     * @return 返回值说明
     * @throws Exception 异常说明
     */
    public String method(String param) throws Exception {
        // 行内注释：说明复杂逻辑
        return result;
    }
}
```

## 常见问题

### 代码坏味道
| 问题 | 表现 | 重构方法 |
|------|------|----------|
| 重复代码 | 多处相同逻辑 | 提取方法/类 |
| 长方法 | 方法超过50行 | 拆分方法 |
| 大类 | 类职责过多 | 拆分类 |
| 魔法数 | 硬编码数字 | 定义常量 |
| 深层嵌套 | if嵌套过深 | 卫语句/策略模式 |

### 反模式
- God Class: 类职责过多
- Spaghetti Code: 代码逻辑混乱
- Copy-Paste: 复制粘贴代码
- Magic Numbers: 魔法数字
