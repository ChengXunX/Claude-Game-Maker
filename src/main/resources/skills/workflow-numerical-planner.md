## 数值设计核心公式

### 1. 成长曲线公式
```
线性成长：value = base + level * growth
指数成长：value = base * (1 + rate)^level
对数成长：value = base + log(level) * factor

选择依据：
- 线性：适合简单系统，易于理解
- 指数：适合后期爆发，有成就感
- 对数：适合前期快速，后期放缓
```

### 2. 伤害计算公式
```
简单公式：damage = attack - defense
复杂公式：damage = attack * (1 - defense / (defense + 100))
暴击公式：damage = base_damage * crit_multiplier

设计要点：
- 防御要有收益递减，避免堆防御无敌
- 暴击要有上限，避免暴击流太强
- 伤害要有下限，避免0伤害
```

### 3. 经济平衡公式
```
货币产出 = 任务奖励 + 掉落 + 活动
货币消耗 = 购买 + 强化 + 解锁
平衡点：产出 ≈ 消耗 * 1.2

设计要点：
- 产出要略大于消耗，让玩家有盈余
- 要有货币沉淀（消耗大于产出的场景）
- 要控制通货膨胀
```

## 常见数值设计

### 1. 等级系统
```
经验值曲线：
Lv1→Lv2: 100 exp
Lv2→Lv3: 200 exp
Lv3→Lv4: 400 exp
...
Lv(n-1)→Lv(n): 100 * 2^(n-2) exp

设计要点：
- 前期升级快，后期升级慢
- 每级奖励要有吸引力
- 要有等级上限
```

### 2. 装备系统
```
装备属性：
基础属性 = 装备等级 * 品质系数
强化属性 = 基础属性 * 强化等级 * 强化系数

品质系数：
普通: 1.0
优秀: 1.2
稀有: 1.5
史诗: 2.0
传说: 3.0

设计要点：
- 品质差距要明显
- 强化要有失败风险
- 要有保底机制
```

### 3. 经济系统
```
货币类型：
- 金币：免费货币，通过游戏获取
- 钻石：付费货币，通过充值获取
- 积分：活动货币，通过活动获取

获取比例：
1钻石 ≈ 100金币 ≈ 10积分

设计要点：
- 货币要分层
- 获取要合理
- 消耗要有价值
```

### 4. 概率设计
```
掉落概率：
普通怪物：5%掉落装备
精英怪物：20%掉落装备
BOSS：100%掉落装备

强化概率：
+1→+2: 100%
+2→+3: 90%
+3→+4: 80%
...
+9→+10: 10%

设计要点：
- 概率要透明
- 要有保底机制
- 要有失败惩罚
```

## 平衡性测试

### 1. 测试方法
```
1. 数值模拟：用代码模拟战斗
2. 人工测试：实际游玩测试
3. 数据分析：分析玩家数据
4. A/B测试：对比不同方案
```

### 2. 关键指标
```
1. 通关率：玩家通关比例
2. 平均时长：每局平均时长
3. 付费率：付费玩家比例
4. 流失率：玩家流失比例
```

### 3. 调整策略
```
1. 数据驱动：根据数据调整
2. 小步快跑：每次调整幅度小
3. 及时响应：发现问题及时调整
4. 保持沟通：和策划、开发沟通
```

## 常见错误

1. **数值膨胀**：数值越来越大，失去意义
2. **平衡性差**：某些组合太强或太弱
3. **概率不透明**：玩家不知道概率
4. **经济崩溃**：通货膨胀严重
5. **成长无感**：升级没有成就感
6. **迭代数值不调整**：要根据反馈调整数值
7. **不参与评审**：要参与评审验证数值
8. **不总结经验**：要及时总结数值设计经验
