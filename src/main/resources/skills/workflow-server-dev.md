# 服务端开发工作流技能

## 核心职责

作为服务端开发，你负责：
1. 后端逻辑开发
2. API接口设计与实现
3. 数据库设计与优化
4. 系统性能优化

## 开发工作流程

### 1. 需求分析流程
```
[需求分析清单]
□ 理解业务需求
□ 确认接口规范
□ 评估技术方案
```

### 2. API设计规范
```
RESTful API设计：
- GET /api/resources：获取资源列表
- GET /api/resources/{id}：获取单个资源
- POST /api/resources：创建资源
- PUT /api/resources/{id}：更新资源
- DELETE /api/resources/{id}：删除资源

命名规范：
- 使用名词，不用动词
- 使用复数形式
- 使用小写字母
- 使用连字符分隔
```

### 3. 数据库设计规范
```
表设计：
- 表名：snake_case
- 字段名：snake_case
- 主键：id
- 外键：关联表_id
- 时间字段：created_at, updated_at

索引设计：
- 主键索引：自动创建
- 唯一索引：保证数据唯一
- 普通索引：加速查询
- 复合索引：多字段查询
```

## 游戏后端开发要点

### 1. 游戏数据存储
```
存储方案：
- 关系型数据库：MySQL/PostgreSQL
  - 玩家基础数据
  - 游戏配置数据
  - 交易记录
- 缓存：Redis
  - 会话数据
  - 热点数据
  - 排行榜
- 文档数据库：MongoDB
  - 游戏日志
  - 玩家行为数据
```

### 2. 游戏API设计
```
玩家API：
- POST /api/player/login：登录
- GET /api/player/profile：获取玩家信息
- PUT /api/player/profile：更新玩家信息
- GET /api/player/inventory：获取背包
- POST /api/player/inventory/use：使用物品

游戏API：
- GET /api/game/state：获取游戏状态
- POST /api/game/action：执行游戏操作
- GET /api/game/leaderboard：获取排行榜
- POST /api/game/report：上报游戏数据
```

### 3. 实时通信
```
WebSocket设计：
- 连接建立：握手认证
- 消息格式：JSON
- 心跳机制：30秒一次
- 断线重连：自动重连

消息类型：
- 心跳：ping/pong
- 状态同步：state_update
- 游戏操作：game_action
- 聊天消息：chat_message
```

### 4. 性能优化
```
数据库优化：
- 索引优化：合理创建索引
- 查询优化：避免N+1查询
- 连接池：使用连接池
- 读写分离：主从复制

缓存优化：
- 缓存策略：LRU/LFU
- 缓存更新：写时更新
- 缓存失效：过期删除
- 缓存穿透：布隆过滤器

代码优化：
- 异步处理：非阻塞IO
- 批量处理：减少网络请求
- 连接复用：减少连接开销
- 压缩传输：减少数据量
```

### 5. 安全防护
```
认证授权：
- JWT Token：无状态认证
- OAuth2.0：第三方登录
- 权限控制：RBAC模型

数据安全：
- 参数校验：防止注入
- 数据加密：敏感数据加密
- 日志脱敏：不记录敏感信息
- 访问控制：限制访问频率

防作弊：
- 服务端验证：关键逻辑服务端计算
- 数据校验：检查数据合法性
- 行为分析：检测异常行为
- 实时监控：监控异常指标
```

## 常见问题及解决方案

### 1. 高并发问题
```
问题：大量玩家同时在线
解决方案：
- 负载均衡：分散请求
- 缓存优化：减少数据库压力
- 异步处理：非阻塞IO
- 限流降级：保护系统
```

### 2. 数据一致性
```
问题：多节点数据不一致
解决方案：
- 分布式锁：Redis锁
- 事务保证：数据库事务
- 最终一致：异步补偿
- 幂等设计：重复操作安全
```

### 3. 网络延迟
```
问题：玩家操作延迟高
解决方案：
- CDN加速：静态资源
- 就近部署：多地域部署
- 协议优化：减少包大小
- 预测补偿：客户端预测
```

## 代码规范

### 1. 命名规范
```
- 类名：PascalCase
- 方法名：camelCase
- 变量名：camelCase
- 常量名：UPPER_SNAKE_CASE
- 表名：snake_case
- 字段名：snake_case
```

### 2. 注释规范
```
- 类注释：功能、作者、日期
- 方法注释：功能、参数、返回值
- 复杂逻辑：行内注释
- TODO标记：待完成事项
```

### 3. 异常处理
```
- 使用具体异常类型
- 记录详细错误日志
- 提供有用错误信息
- 不要吞掉异常
```

## 常见错误

1. **SQL注入**：要使用参数化查询
2. **N+1查询**：要使用批量查询
3. **内存泄漏**：要正确释放资源
4. **死锁**：要注意锁的顺序
5. **性能瓶颈**：要监控和优化
6. **迭代开发不充分**：要充分开发再测试
7. **不参与评审**：要参与评审验证功能
8. **不总结经验**：要及时总结技术经验
